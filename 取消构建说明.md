# 如何取消GitHub Actions构建

## 方法1: 通过网页界面

1. 访问Actions页面：
   https://github.com/diskk-create/wangzhe-autoclicker/actions

2. 找到正在运行的workflow

3. 点击进入详情页

4. 点击右上角的 **"Cancel workflow"** 按钮

## 方法2: 使用GitHub CLI

如果您安装了GitHub CLI，可以运行：

```powershell
# 列出正在运行的workflow
gh run list --repo diskk-create/wangzhe-autoclicker --status in_progress

# 取消指定的workflow（替换RUN_ID）
gh run cancel RUN_ID --repo diskk-create/wangzhe-autoclicker
```

## 方法3: 禁用workflow

如果不想继续构建，可以禁用workflow：

1. 访问：https://github.com/diskk-create/wangzhe-autoclicker/settings/actions

2. 在 "Actions permissions" 部分：
   - 选择 "Disable actions"
   - 或选择 "Only allow select actions"

## 暂时停止自动构建

如果只想暂时停止，可以修改workflow文件：

### 方法1: 重命名workflow文件

```powershell
cd C:\Users\Administrator\.openclaw\workspace\wangzhe_autoclicker_v3
mv .github/workflows/build.yml .github/workflows/build.yml.disabled
git add .
git commit -m "禁用自动构建"
git push
```

### 方法2: 添加触发条件

修改 `.github/workflows/build.yml`，添加手动触发：

```yaml
on:
  # push:
  #   branches: [ main, master ]
  workflow_dispatch:  # 只允许手动触发
```
