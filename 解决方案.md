# APK闪退问题 - 最终解决方案

## 问题总结

**成功构建版本**（commit 4de58c6）虽然构建成功，但运行时闪退。

**根本原因**：
1. 代码依赖`android`模块和`mActivity`
2. 缺少Java无障碍服务实现
3. 无障碍服务需要正确的AndroidManifest.xml配置

## 解决方案对比

| 方案 | 文件 | 构建配置 | 优点 | 缺点 | 推荐度 |
|------|------|---------|------|------|--------|
| ADB方式 | main_simple.py | buildozer_simple.spec | 简单、稳定、可用 | 需要ADB连接 | ⭐⭐⭐⭐ |
| Pyjnius | main_pyjnius.py | buildozer_pyjnius.spec | 可在APK中运行 | 需要ROOT或特殊权限 | ⭐⭐⭐⭐⭐ |
| 无障碍服务 | main.py | buildozer.spec | 功能完整 | 需要Java代码，复杂 | ⭐⭐ |

## 推荐方案：Pyjnius版本

### 为什么推荐Pyjnius版本？
1. ✅ **可以构建成功** - 只依赖python3,kivy,pyjnius
2. ✅ **可以在APK中运行** - 不需要ADB连接
3. ✅ **代码简单** - 不需要Java代码
4. ✅ **使用Runtime.exec** - 直接执行shell命令

## 三种方案详解

### 方案1: ADB方式（main_simple.py）

**适用场景**：
- 模拟器环境
- 有ADB连接
- 开发测试

**特点**：
- 使用subprocess调用ADB命令
- 不依赖Android API
- 构建最简单

**使用方法**：
```bash
cp main_simple.py main.py
cp buildozer_simple.spec buildozer.spec
# 构建并安装
```

**ADB命令**：
```bash
# 连接模拟器
adb connect 127.0.0.1:21503

# 测试点击
adb shell input tap 640 360
```

### 方案2: Pyjnius版本（main_pyjnius.py）⭐ 推荐

**适用场景**：
- 真机运行
- 独立APK
- 不需要ADB

**特点**：
- 使用Runtime.exec执行shell命令
- 需要ROOT或特殊权限
- 构建简单

**实现原理**：
```python
# 使用Java Runtime执行shell命令
Runtime = autoclass('java.lang.Runtime')
runtime = Runtime.getRuntime()
process = runtime.exec("input tap 640 360")
process.waitFor()
```

**权限说明**：
- 普通设备：需要ROOT权限执行input命令
- 模拟器：通常默认有权限
- 部分设备：可以通过无障碍服务实现

### 方案3: 完整无障碍服务（main.py）

**适用场景**：
- 需要完整功能
- 需要截屏识别
- 生产环境

**需要**：
1. Java无障碍服务类
2. AndroidManifest.xml配置
3. res/xml/accessibility_service_config.xml
4. 用户手动开启无障碍服务

**实现步骤**：
1. 创建Java服务类
2. 配置AndroidManifest
3. 创建配置XML
4. 处理权限请求
