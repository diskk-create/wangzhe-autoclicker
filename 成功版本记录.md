## 技术特点

### 为什么不会闪退？

**关键差异**：
```python
# ❌ 旧版本（会闪退）
from android import mActivity  # 需要Java无障碍服务代码

# ✅ 新版本（不会闪退）
from jnius import autoclass
PythonActivity = autoclass('org.kivy.android.PythonActivity')
mActivity = PythonActivity.mActivity  # 不需要额外Java代码
```

### 点击实现
```python
# 使用Runtime.exec执行shell命令
Runtime = autoclass('java.lang.Runtime')
runtime = Runtime.getRuntime()
process = runtime.exec(f"input tap {x} {y}")
process.waitFor(1, TimeUnit.SECONDS)
```

### 权限说明
- 正常运行：不需要ROOT
- 点击功能：可能需要ROOT或特殊权限
- 图像识别：不需要特殊权限

## 后续修改指南

### 修改按钮坐标
编辑 `main.py` 中的 `self.buttons` 字典：
```python
self.buttons = {
    'button_name': {'x': 新x坐标, 'y': 新y坐标, 'desc': '描述'},
    ...
}
```

### 修改流程步骤
编辑 `main.py` 中的 `self.flow_steps` 列表：
```python
self.flow_steps = [
    ('button_name', 等待秒数, '步骤描述'),
    ...
]
```

### 修改识别阈值
编辑 `main.py` 中的 `ImageMatcher` 初始化：
```python
self.matcher = ImageMatcher(template_dir='templates', threshold=0.9)  # 0.9 = 90%
```

### 添加新模板
1. 准备模板图片（PNG格式）
2. 放入 `templates/` 目录
3. 在 `ImageMatcher._load_templates()` 中添加映射

## 已知问题

### 1. 点击功能需要权限
- **问题**：普通设备可能无法执行 `input tap` 命令
- **解决**：
  - ROOT设备：授予权限
  - 模拟器：通常默认有权限
  - 普通设备：实现无障碍服务（需要Java代码）

### 2. 图像识别需要截图
- **问题**：当前版本未实现截图功能
- **解决**：
  - 使用MediaProjection API（需要权限）
  - 或使用ADB截图

### 3. 模板文件缺失
- **问题**：`templates/` 目录需要包含模板图片
- **解决**：
  - 手动准备模板图片
  - 或使用坐标点击作为兜底方案
