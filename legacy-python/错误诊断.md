# GitHub Actions 构建错误诊断

## 🔍 可能的错误原因

### 1. 缺少必要文件

检查项目根目录是否有以下文件：
- [x] main.py ✅
- [x] buildozer.spec ✅
- [x] scripts/ 目录 ✅
- [x] assets/configs/ 目录 ✅

### 2. buildozer.spec 配置问题

可能需要添加的配置：
- source.dir 需要指向正确的目录
- requirements 需要指定正确的依赖

### 3. Python版本问题

构建使用Python 3.10，但可能需要调整。

---

## 🛠️ 快速修复方案

### 方案1: 添加 buildozer 初始化

创建一个简单的测试版本，先确保构建流程正常：

```yaml
# 修改 .github/workflows/build.yml
- name: Initialize buildozer
  run: |
    buildozer init || true
    buildozer android debug
```

### 方案2: 使用官方 buildozer action

```yaml
- name: Build with buildozer
  uses: ArtemSBulgakov/buildozer-action@v1
  id: build
  with:
    command: android debug
```

## 🔧 常见错误及解决方案

### 错误1: "No main.py found"
**解决**: 确保main.py在项目根目录

### 错误2: "buildozer.spec not found"
**解决**: 确保buildozer.spec在项目根目录

### 错误3: "requirements not found"
**解决**: 检查buildozer.spec中的requirements字段

### 错误4: "Android SDK/NDK download failed"
**解决**: 使用缓存或增加超时时间

### 错误5: "Build timeout"
**解决**: 增加timeout-minutes
