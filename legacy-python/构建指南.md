## 🚀 构建方法

### 方法1: GitHub Actions（最简单）⭐

**时间**: 30分钟（包括首次设置）

**步骤**:
1. 创建GitHub仓库
2. 上传代码
3. 等待自动构建
4. 下载APK

**优点**:
- ✅ 无需本地环境
- ✅ 自动构建
- ✅ 完全免费
- ✅ 持续集成

**详细步骤**:

```powershell
# 1. 进入项目目录
cd C:\Users\Administrator\.openclaw\workspace\wangzhe_autoclicker_v3

# 2. 初始化Git
git init
git add .
git commit -m "v3.0: 完整重构版本"

# 3. 设置分支
git branch -M main

# 4. 添加远程仓库（替换YOUR_USERNAME）
git remote add origin https://github.com/YOUR_USERNAME/wangzhe-autoclicker.git

# 5. 推送代码
git push -u origin main

# 6. 访问GitHub，等待构建完成
# https://github.com/YOUR_USERNAME/wangzhe-autoclicker/actions

# 7. 下载APK
# 在Actions页面下载Artifacts
```

### 方法2: WSL本地构建

**时间**: 40-60分钟（首次）

**前置要求**:
- Windows 10/11
- WSL2
- Ubuntu

**步骤**:

```bash
# 1. 安装WSL（Windows PowerShell管理员模式）
wsl --install -d Ubuntu

# 2. 进入WSL
wsl

# 3. 安装依赖
sudo apt update
sudo apt install -y git zip unzip openjdk-17-jdk \
    autoconf libtool pkg-config \
    libncurses5-dev libncursesw5-dev libtinfo5 \
    cmake libffi-dev libssl-dev automake

# 4. 安装Python依赖
pip3 install buildozer cython

# 5. 进入项目目录
cd /mnt/c/Users/Administrator/.openclaw/workspace/wangzhe_autoclicker_v3

# 6. 构建
buildozer android debug

# 7. 查找APK
ls -lh bin/*.apk
```

**优点**:
- ✅ 本地调试
- ✅ 快速迭代
- ✅ 完全控制

**缺点**:
- ⚠️ 需要配置环境
- ⚠️ 首次下载慢

---

### 方法3: Docker构建

**时间**: 20-30分钟

**前置要求**:
- Docker Desktop

**步骤**:

```powershell
# 1. 拉取buildozer镜像
docker pull kivy/buildozer

# 2. 进入项目目录
cd C:\Users\Administrator\.openclaw\workspace\wangzhe_autoclicker_v3

# 3. 构建APK
docker run --rm -v "${PWD}":/home/user/hostcwd kivy/buildozer android debug

# 4. 查找APK
dir bin\*.apk
```

**优点**:
- ✅ 环境隔离
- ✅ 快速开始

**缺点**:
- ⚠️ 需要Docker
- ⚠️ 权限可能有问题

## 🎯 架构选择

编辑 `buildozer.spec` 第79行：

### 真机专用（推荐）
```ini
android.archs = armeabi-v7a, arm64-v8a
```
**APK大小**: ~30MB
**构建时间**: ~20分钟
**兼容性**: 所有ARM真机

### 模拟器专用
```ini
android.archs = x86, x86_64
```
**APK大小**: ~35MB
**构建时间**: ~20分钟
**兼容性**: x86模拟器

### 全架构（最大兼容）
```ini
android.archs = armeabi-v7a, arm64-v8a, x86, x86_64
```
**APK大小**: ~80MB
**构建时间**: ~40分钟
**兼容性**: 真机 + 模拟器

## 🐛 常见问题

### Q1: 构建失败 - 下载超时
```bash
# 重新触发构建
buildozer android clean
buildozer android debug
```

### Q2: ADB找不到设备
```powershell
# 重启ADB服务
adb kill-server
adb start-server
adb devices
```

### Q3: 安装失败 - 签名冲突
```powershell
# 卸载旧版本
adb uninstall org.wangzhe.wangzheautoclicker

# 重新安装
adb install bin\wangzheautoclicker-3.0.0-debug.apk
```

### Q4: 启动崩溃
```powershell
# 查看详细日志
adb logcat -c
adb shell am start -n org.wangzhe.wangzheautoclicker/org.kivy.android.PythonActivity
adb logcat | Select-String "python|FATAL|Exception"
```
