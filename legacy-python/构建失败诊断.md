# 构建失败诊断指南

## 查看构建失败原因

### 方法1: 通过GitHub网页查看

1. 访问Actions页面：
   https://github.com/diskk-create/wangzhe-autoclicker/actions

2. 点击最新失败的workflow（红色叉号）

3. 点击 "Build Android APK" 任务

4. 展开各个步骤，查看失败的具体原因

5. 常见失败步骤：
   - "Install system dependencies" - 系统依赖安装失败
   - "Install buildozer" - buildozer安装失败
   - "Build APK" - APK构建失败（最常见）

## 常见构建失败原因

### 1. buildozer.spec 配置错误
**症状**: "Build APK" 步骤失败
**原因**: 配置项格式错误或依赖问题

**解决方案**:
- 检查 `requirements` 格式
- 简化依赖列表
- 移除可能失败的配置

### 2. 系统依赖缺失
**症状**: "Install system dependencies" 失败
**原因**: apt包不可用或网络问题

**解决方案**:
- 重试构建
- 修改依赖列表

### 3. Python依赖冲突
**症状**: "Install buildozer" 失败
**原因**: Python包版本冲突

**解决方案**:
- 指定包版本
- 使用缓存

### 4. NDK/SDK下载失败
**症状**: "Build APK" 步骤长时间运行后失败
**原因**: 下载超时或网络问题

**解决方案**:
- 使用缓存
- 增加超时时间

## 快速修复方案

### 方案1: 使用更简单的配置

创建最小化的 buildozer.spec：

```ini
[app]
title = 王者荣耀自动点击器
package.name = wangzheautoclicker
package.domain = org.wangzhe
source.dir = .
source.include_exts = py,png,jpg,kv,atlas,json,txt
version = 3.0.0
requirements = python3,kivy
orientation = portrait
fullscreen = 0
android.permissions = SYSTEM_ALERT_WINDOW
android.api = 31
android.minapi = 21
android.ndk = 25b
android.skip_update = False
android.accept_sdk_license = True
android.archs = armeabi-v7a,arm64-v8a

[buildozer]
log_level = 2
warn_on_root = 1
```

### 方案2: 使用官方buildozer action

修改 `.github/workflows/build.yml`：

```yaml
- name: Build with buildozer
  uses: ArtemSBulgakov/buildozer-action@v1
  id: build
  with:
    command: android debug
```

### 方案3: 分步骤构建

修改workflow，分步骤执行，更容易定位问题：

```yaml
- name: Initialize buildozer
  run: |
    buildozer init || true

- name: Download dependencies
  run: |
    buildozer android debug || true

- name: Build APK
  run: |
    buildozer android debug
```
